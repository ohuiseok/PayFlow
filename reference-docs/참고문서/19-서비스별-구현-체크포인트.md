# 서비스별 구현 체크포인트

이 문서는 블로그 작성 시 각 서비스에서 어떤 코드를 보면 되는지 정리한 가이드입니다.

## api-gateway

읽을 코드:

```text
api-gateway/src/main/java/com/payflow/gateway/filter/JwtAuthenticationFilter.java
api-gateway/src/main/java/com/payflow/gateway/config/SecurityConfig.java
api-gateway/src/main/resources/application.yml
```

설명할 내용:

- JWT 검증
- 인증 제외 경로
- 외부 사용자 헤더 제거
- 내부 사용자 헤더 주입
- 서비스 route

검증 포인트:

- token 없는 요청 거절
- 외부 `X-User-*` 조작 방어
- 정상 token 요청 라우팅

## wallet-service

읽을 코드:

```text
wallet-service/src/main/java/com/payflow/wallet/controller/WalletController.java
wallet-service/src/main/java/com/payflow/wallet/service/WalletService.java

```

설명할 내용:

- 잔액의 단일 진실 공급원
- 입금/출금 transaction
- 잔액 변경 이력
- reference 기반 중복 방어

- 잔액 부족 시 실패
- 거래 이력과 balance 일치

## banking-service

읽을 코드:

```text
banking-service/src/main/java/com/payflow/banking/controller/BankingController.java
banking-service/src/main/java/com/payflow/banking/service/BankingService.java
banking-service/src/main/java/com/payflow/banking/openbanking/RealOpenBankingClient.java
banking-service/src/main/java/com/payflow/banking/service/TokenCryptoService.java
banking-service/src/main/java/com/payflow/banking/entity/BankingTransfer.java
```

설명할 내용:

- 계좌 등록
- 계좌번호 masking/hash
- Open Banking token 암호화
- 충전/출금 상태 모델
- result-check
- compensation

검증 포인트:

- 같은 idempotency key 처리
- 모호한 외부 응답 상태 저장
- 지갑 반영은 은행 성공 이후 수행
- no-permission API를 성공 처리하지 않음

## transfer-service

읽을 코드:

```text
transfer-service/src/main/java/com/payflow/transfer/controller/TransferController.java
transfer-service/src/main/java/com/payflow/transfer/service/TransferService.java
transfer-service/src/main/java/com/payflow/transfer/lock/RedisDistributedLock.java
transfer-service/src/main/java/com/payflow/transfer/outbox/OutboxEvent.java

transfer-service/src/main/java/com/payflow/transfer/controller/OutboxMonitoringController.java
```

설명할 내용:

- 송금 상태 머신
- idempotency key와 request hash
- Redis sender wallet lock
- sender withdraw, receiver deposit
- compensation refund
- transactional outbox
- outbox monitoring

검증 포인트:

- 같은 송금 요청 재시도
- 출금 후 입금 실패
- 보상 환불 중복 방지
- outbox retry/stale recovery

## 블로그 작성 팁

각 서비스 글은 아래 구조를 반복하면 읽기 좋습니다.

```text
1. 이 서비스가 맡은 책임
2. 이 서비스가 소유한 데이터
3. 핵심 API
4. 가장 중요한 도메인 규칙
5. 실패 처리
6. 테스트한 것
7. 다음 서비스와 이어지는 지점
```
