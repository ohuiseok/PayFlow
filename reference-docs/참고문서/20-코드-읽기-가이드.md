# 코드 읽기 가이드

> 도메인 전환 안내: 현재 PayFlow는 **청년 정책 참여 미션 및 지원금 지급 플랫폼**으로 설명한다. 내부 구현 호환성을 위해 `PARENT`/`CHILD`, `/api/families`, `/api/missions`, `/api/cashbook`, `reward-service` 같은 명칭은 유지하지만, 문서와 발표에서는 각각 **기관 담당자**, **청년 참여자**, **참여자 연결**, **정책 미션**, **지원금 사용 내역**, **정책 미션/지원금 서비스**로 해석한다.

이 문서는 블로그 독자나 리뷰어가 PayFlow 코드를 어떤 순서로 읽으면 좋은지 안내합니다.

## 전체 코드 읽기 순서

### 1. 설계 문서

먼저 서비스 목적과 흐름을 봅니다.

```text
docs/payflow-service-planning.md
docs/service-flow.md
reference-docs/참고문서/00-index.md
reference-docs/참고문서/17-블로그-발행-로드맵.md
```

### 2. Docker Compose와 route

서비스가 어떻게 연결되는지 봅니다.

```text
docker-compose.yml
infrastructure/nginx/nginx.conf
api-gateway/src/main/resources/application.yml
```

확인할 것:

- 서비스 포트
- DB 이름
- Redis/Kafka 연결
- Gateway route

### 3. 공통 보안 흐름

```text
api-gateway/src/main/java/com/payflow/gateway/filter/JwtAuthenticationFilter.java
*/config/GatewayRequestFilter.java
```

확인할 것:

- Gateway가 어떤 헤더를 제거하는가
- 어떤 헤더를 내부 서비스에 전달하는가
- 하위 서비스가 internal secret을 검증하는가

### 4. user -> wallet 기본 흐름

```text
user-service/src/main/java/com/payflow/user/service/UserService.java
wallet-service/src/main/java/com/payflow/wallet/service/WalletService.java
```

확인할 것:

- 회원가입 후 지갑 생성
- 사용자별 지갑 unique
- 입금/출금 transaction

### 5. 멱등성 흐름

```text
transfer-service/src/main/java/com/payflow/transfer/service/TransferService.java
banking-service/src/main/java/com/payflow/banking/service/BankingService.java
wallet-service/src/main/java/com/payflow/wallet/service/WalletService.java
```

확인할 것:

- `Idempotency-Key`
- `requestHash`
- `referenceType`
- `referenceId`

### 6. 송금과 보상

```text
transfer-service/src/main/java/com/payflow/transfer/service/TransferService.java
transfer-service/src/main/java/com/payflow/transfer/controller/TransferController.java
```

확인할 것:

- 송금 상태 전이
- Redis lock
- sender withdraw
- receiver deposit
- compensation refund

### 7. Outbox와 Kafka

```text
transfer-service/src/main/java/com/payflow/transfer/outbox/OutboxEvent.java
transfer-service/src/main/java/com/payflow/transfer/outbox/OutboxEventRelay.java
ledger-service/src/main/java/com/payflow/ledger/event/TransferEventConsumer.java
```

확인할 것:

- outbox 저장 시점
- `PENDING`, `PROCESSING`, `PUBLISHED`, `FAILED`
- retry count
- stale processing recovery
- ledger consumer dedup

### 8. Reward flow

```text
reward-service/src/main/java/com/payflow/reward/service/RewardService.java
reward-service/src/main/java/com/payflow/reward/controller/MissionController.java
reward-service/src/main/java/com/payflow/reward/controller/FamilyController.java
```

확인할 것:

- parent/child 권한
- mission status
- approval
- pay
- transfer-service 위임

### 9. Banking/Open Banking

```text
banking-service/src/main/java/com/payflow/banking/service/BankingService.java
```

확인할 것:

- token 암호화

### 10. Frontend

```text
sample-react/src/api
sample-react/src/screens
sample-react/src/hooks
sample-react/e2e
```

확인할 것:

- 화면별 API 호출
- parent approval에서 approve + pay
- dummy mode와 API mode
- error/loading state

## 코드 리뷰 질문

읽으면서 아래 질문에 답할 수 있으면 프로젝트 이해가 된 것입니다.

- 잔액은 어느 서비스가 변경하는가?
- 같은 송금 요청이 두 번 오면 어디서 막히는가?
- sender 출금 후 receiver 입금이 실패하면 어떤 상태가 되는가?
- Kafka 발행 실패는 어떻게 복구하는가?
- ledger-service가 같은 이벤트를 두 번 받으면 어떻게 되는가?
- Open Banking timeout은 실패인가, 처리 중인가?
- 화면에서 보상 지급은 어떤 API들을 호출하는가?
