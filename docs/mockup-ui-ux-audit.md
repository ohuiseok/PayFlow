> 도메인 전환 안내: 현재 PayFlow는 **청년 정책 참여 미션 및 지원금 지급 플랫폼**으로 설명한다. 내부 구현 호환성을 위해 `PARENT`/`CHILD`, `/api/families`, `/api/missions`, `/api/cashbook`, `reward-service` 같은 명칭은 유지하지만, 문서와 발표에서는 각각 **기관 담당자**, **청년 참여자**, **참여자 연결**, **정책 미션**, **지원금 사용 내역**, **정책 미션/지원금 서비스**로 해석한다.

점검 기준일: 2026-06-15

## MVP 화면 기준

포트폴리오 제출용 화면은 결제 MVP 흐름을 설명하는 데 필요한 화면만 남긴다.

핵심 흐름:

1. 로그인
2. 회원가입과 역할 선택
3. 기관-청년 참여자 연결
4. 기관 홈
5. 기관 지갑 충전
6. 미션 생성
7. 청년 홈
8. 미션 제출
9. 기관 승인/반려
10. 보상 지급 결과
11. 연결 계좌 등록

## 통합 처리

사용 기록 상세는 청년 홈의 최근 돈 기록으로 표현한다.

충전 완료/실패는 충전 화면의 상태 메시지로 표현한다.

참여자 연결 완료는 연결 화면의 성공 상태로 표현한다.

미션 상세 상태는 청년 홈과 미션 제출 화면에서 표현한다.

## API 영향

UI는 `docs/api-spec.md`의 MVP API만 사용한다.

프론트에서 별도 파일 업로드, 알림, 설정, 정산, 출금 화면을 전제로 두지 않는다.

미션 반려 후 다시 제출하려면 같은 `PATCH /api/missions/{missionId}/submit` 흐름을 사용한다.
